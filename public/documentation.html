Welcome to myFlix Documentation

Base URL: http://localhost:8080

This documentation provides an overview of the myFlix application, its features, and how to use it.

Installation

In order to use the App you need to download the files from GitHub repository and install the necessary dependencies by running npm install in the project directory.

Features

  • Return a list of ALL movies to the user
  • Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user
  • Return data about a genre (description) by name/title (e.g., “Thriller”)
  • Return data about a director (bio, birth year, death year) by name
  • Allow new users to register
  • Allow users to update their user info (username, password, email, date of birth)
  • Allow users to add a movie to their list of favorites
  • Allow users to remove a movie from their list of favorites
  • Allow existing users to deregister

API Endpoints

  • /movies
  • /movies/[title]
  • /movies/genre/[genreName]
  • /directors/[name]
  • /users
  • /users/[id]
  • /users/[username]/[favorites]

POST /users

Description:
Adds a new user to the system with the given name.

HTTP Method: POST

URL: http://localhost:8080/users

Path Parameters: None

Request Body: application/json

Example Request:


            {
                "name": 'Kate'      
            }

Success Response:

  • Status Code: 200 OK
  • Content-Type: application/json

Example Response:


            {
                "name": 'Kate',
                "id": 3,
                "favorites": []       
            }

Error Responses:

  • 400 Bad Request – User name is required

PUT /users/:id

Description:
Updates an existing user's name in the system with the given data based on user's ID.

HTTP Method: PUT

URL: http://localhost:8080/users/:id

Path Parameters: id (User ID)

Request Body: application/json

Example Request:


            {
                "name": 'John'      
            }

Success Response:

  • Status Code: 200 OK
  • Content-Type: application/json

Example Response:


            {
                "name": 'John',
                "id": 1,
                "favorites": []       
            }

Error Responses:

  • 400 Bad Request – User not found

PUT /users/:id/:movieTitle

Description:
Updates favorite movies of the user in the system with the given data based on user's ID.

HTTP Method: PUT

URL: http://localhost:8080/users/:id/:movieTitle

Path Parameters: id (User ID), movieTitle (Movie Title)

Request Body: None

Success Response:

  • Status Code: 200 OK
  • Content-Type: String

Example Response:

Favorite movies for user 2 updated successfully by adding Pulp Fiction

Error Responses:

  • 400 Bad Request – User not found

PUT /users/:id/:movieTitle

Description:
Updates favorite movies of the user in the system with the given data based on user's ID.

HTTP Method: DELETE

URL: http://localhost:8080/users/:id/:movieTitle

Path Parameters: id (User ID), movieTitle (Movie Title)

Request Body: None

Success Response:

  • Status Code: 200 OK
  • Content-Type: String

Example Response:

Favorite movies for user 2 updated successfully by removing Pulp Fiction

Error Responses:

  • 400 Bad Request – User not found
